# AI Guiding System wx

一个基于微信小程序的 AI 导览系统原型项目，面向博物馆/展馆导览场景，演示“身份标签 + 馆内地图 + 自动讲解 + 分屏导览”的基础交互流程。

## 项目简介

项目以“展馆智能导览助手”为核心概念，当前版本聚焦前端原型验证，包含两页流程：

1. 用户先在首页设置自己的导览身份标签。
2. 进入导览页后，可体验地图、今日馆藏、快速路线、亲子问答等入口。
3. 通过模拟定位进入展区后，可触发文物讲解卡片。
4. 讲解内容会根据用户身份标签动态切换。

当前项目不依赖后端服务，展区数据、讲解文案、路线内容均为前端内置示例数据，适合用于课程作业、产品原型展示和交互验证。

## 功能特性

- 身份标签设置：支持手动输入或快捷选择“历史极客”“亲子家庭”“艺术爱好者”“摄影打卡党”“轻松逛馆”等标签。
- 个性化讲解：进入导览页后，根据用户标签展示不同风格的文物讲解文案。
- 馆内分屏导览：将底部功能入口拖拽到页面中央，可打开分屏内容。
- 模拟馆内定位：在地图模式下可启动定位模拟，沿预设路径移动。
- 展区触发讲解：用户进入指定展区后自动弹出讲解卡片。
- 语音讲解预留：已预留音频播放逻辑，配置音频地址后可扩展自动语音讲解。

## 交互流程

### 1. 身份设置页

路径：`pages/profile/index`

- 输入或选择身份标签
- 点击“开始导览”
- 标签会写入 `wx.setStorageSync("userTag")`

### 2. 导览页

路径：`pages/guide/index`

- 支持从身份页跳转进入
- 页面底部提供 4 个可拖拽导览入口：
  - 地图
  - 今日馆藏
  - 快速路线
  - 亲子问答
- 拖拽到页面中央时，打开对应分屏内容
- 在“馆内地图”模式下可以：
  - 启动模拟定位
  - 停止模拟
  - 重置位置
- 当模拟位置进入 `relicA` 展区时，会触发“文物A自动讲解”

## 关键实现说明

### 个性化讲解

导览页中的 `GUIDE_TEXT` 根据用户标签映射不同讲解文案，默认使用 `default` 文案兜底。

### 分屏拖拽交互

底部操作入口使用 `movable-area` + `movable-view` 实现。拖拽结束后会计算当前入口中心点与页面中央目标区域的距离，命中后打开分屏内容。

### 自动讲解触发

地图中预设了多个展区矩形区域，用户位置进入 `relicA` 后会触发讲解卡片，并调用语音播放逻辑。

### 语音讲解扩展

当前 `pages/guide/index.js` 中：

```js
const GUIDE_AUDIO_URL = "";
```

将其替换为可访问的音频地址后，即可启用自动语音讲解能力。

## 当前限制

- 当前为前端原型，未接入真实 AI 服务。
- 当前为示例馆内地图，定位点与展区坐标均为静态数据。
- 当前未接入真实室内定位能力。
- 当前未配置讲解音频资源，默认使用文字讲解。
- 今日馆藏、快速路线、亲子问答内容仍为静态演示文案。
